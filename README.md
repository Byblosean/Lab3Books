## Мета роботи
Дослідити способи збереження даних у мобільних застосунках Android та отримати практичні навички роботи зі сховищами даних.

---

## Завдання
Доповнити застосунок, створений у лабораторній роботі №2, можливістю збереження даних.

Програма повинна:

- записувати результат взаємодії з інтерфейсом у сховище;
- повідомляти користувача про успішність запису;
- містити кнопку **«Відкрити»**;
- відкривати нову Activity для перегляду вмісту сховища;
- відображати повідомлення, якщо сховище порожнє.

---

## Опис застосунку
У застосунку реалізовано вибір автора книги та року видання з можливістю збереження цих даних у внутрішнє файлове сховище Android.

### Перший фрагмент (`InputFragment`)
Містить:

- `Spinner` для вибору автора;
- `RadioGroup` для вибору року видання;
- кнопку **OK** для підтвердження введення;
- кнопку **«Відкрити»** для перегляду збережених даних.

Після натискання кнопки **OK**:

- обрані дані зберігаються у файл внутрішнього сховища;
- користувач отримує повідомлення про успішне збереження;
- відкривається фрагмент результату.

---

### Другий фрагмент (`ResultFragment`)
Містить:

- `TextView` для відображення обраного автора та року видання;
- кнопку **Cancel**, яка:
  - повертає користувача до першого фрагмента;
  - очищає введені дані.

---

### Activity перегляду даних (`StorageActivity`)
Ця Activity відкривається після натискання кнопки **«Відкрити»**.

Вона:

- читає всі записи з файлу внутрішнього сховища;
- відображає їх у `TextView`.

## Структура застосунку

Застосунок складається з таких компонентів:

- `MainActivity` — контейнер для фрагментів;
- `InputFragment` — введення даних;
- `ResultFragment` — відображення результату;
- `StorageActivity` — перегляд збережених даних;
- `StorageHelper` — клас для роботи з файловим сховищем;
- `SharedViewModel` — передача даних між фрагментами.

---

## Принцип роботи

1. Після запуску застосунку відкривається `InputFragment`.
2. Користувач обирає автора книги зі `Spinner`.
3. Користувач обирає рік видання через `RadioGroup`.
4. Після натискання кнопки **OK**:
   - дані записуються у файл внутрішнього сховища;
   - користувачу відображається повідомлення про успішне збереження;
   - відкривається `ResultFragment`.
5. У `ResultFragment` відображається вибраний автор та рік видання.
6. Кнопка **Cancel** повертає користувача до форми введення та очищає дані.
7. Кнопка **«Відкрити»** відкриває `StorageActivity`.
8. У `StorageActivity` відображається весь вміст файлу зі збереженими даними.
